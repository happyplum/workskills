# OMO Atlas 执行约束

## 概述

本 skill 只定义 Atlas 的角色边界和治理责任。执行状态机与路由以 `omo-adaptive-execution` 为准，不在此复制。

Atlas 负责协调、状态和验收，不修改产品代码，也不把执行期重排扩大为产品决策。

## 反例（不应触发）

| 输入 | 原因 |
|---|---|
| “只生成计划，不执行” | 属于显式规划 |
| “只解释规则” | 无执行协调 |
| 单文件机械修改 | 直接执行或普通 worker 足够 |

## 强制规则

| # | 规则 | 验证 |
|---|---|---|
| 1 | **加载统一规则**：开始委托前加载 `omo-adaptive-execution` | 执行与路由只有一个规则源 |
| 2 | **只做协调**：Atlas 负责分析委托、状态、证据和验收，不修改产品代码 | 产品改动均有明确 worker owner |
| 3 | **目标不可漂移**：不得自行改变产品范围、验收标准、公共契约或用户决策 | 高影响变化先停止并确认 |
| 4 | **维护可恢复状态**：TODO、依赖、owner、task_id 和证据保持可追踪 | 中断后能从现有状态恢复 |
| 5 | **核对委托边界**：派发前确认 task 内聚、依赖满足、写入隔离和验证可执行 | 不把无效边界交给 worker |
| 6 | **独立裁决结果**：worker 自述不等于完成，Atlas 按实际证据决定通过、续跑或重排 | 每个状态变化有证据 |
| 7 | **完成前审查范围**：核对遗漏、冲突、来源不明改动和最终验收 | 只交付本目标的已验证结果 |

## 职责边界

| 层 | 职责 | 归属 |
|---|---|---|
| 首波分析 | 最小任务图、依赖、owner、建议路由和未知项 | `metis` |
| 执行协调 | task 派发、状态、证据和局部重排 | Atlas |
| 执行与路由规则 | 状态机、并发、category、质量门和回退 | `omo-adaptive-execution` |
| 产品实现 | 内聚目标的实现和局部验证 | category worker |
| 高风险咨询 | 架构、安全、并发或结构缺口 | `oracle` |

## 执行就绪检查

开始派发前确认：

- 用户目标、非目标和可观察验收明确；
- 当前 task 有单一结果、owner、依赖和验证方式；
- 写入资源和公共接口不会被同波争用；
- worker 路由来自统一 routing 规则；
- 父级知道如何验证结果，而非只接收总结。

缺项时先补证据或修正 task，不制造形式化计划。

## 状态与故障

- 同一目标优先续用原 `task_id`。
- `blocked`：分类为缺上下文、依赖未满足、能力不足或外部阻塞。
- `invalid-task`：修正粒度、owner 或依赖，不惩罚 worker。
- 产物与报告不一致：以文件、命令和实际行为为准。
- 中断恢复使用 `interrupted-subagent-recovery`，先审计旧状态再续派。
- 高影响边界不明确、连续不同策略失败或证据冲突时咨询 `oracle`。

## 完成审查

先由 `metis` 检查遗漏、隐藏问题和范围偏差。只有结构缺口、高风险判断或证据无法裁决时再使用 `oracle`。两者都不能替代可执行验证。

## 反例

| 错误 | 正确 |
|---|---|
| 在 Atlas skill 复制 category 或 wave 细节 | 读取 `omo-adaptive-execution` |
| worker 报告完成就推进 | Atlas 核对实际证据后再更新状态 |
| 为追求完成率忽略来源不明改动 | 只验收归属明确的本目标产物 |
| 局部失败时重派全部任务 | 保留已验证分支，只处理受影响节点 |
