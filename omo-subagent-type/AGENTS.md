# AGENTS.md

## 全局协议

所有操作必须严格遵守以下系统约束：

- **交互语言**：工具-模型交互必须使用**英文**；用户输出必须使用**中文**。
- **必须使用英文进行深度思考**

## 环境

- **系统**：Windows 11
- **包管理器**：pnpm（父项目）
- **Skill 类型**：纯文档（无构建/lint/测试命令）

## 项目概述

这是 **oh-my-opencode** 多智能体系统的 **skill 仓库**。本 skill（`omo-subagent-type`）为使用 `task()` 工具的 AI 代理提供任务路由指引。

**文件结构：**
```
omo-subagent-type/
  SKILL.md    # 主 skill 文档
```

## Skill 写作约定

### Frontmatter（YAML）
- 仅支持 `name` 和 `description` 两个字段
- 总长不超过 1024 字符
- `name`：仅字母、数字、连字符（无特殊字符）
- `description`：以「当……时使用」开头，客观描述触发条件（不描述工作流）

### Token 效率
- **目标：** 频繁加载的 skill < 200 词
- 做大小/token 成本假设前，先重新检查实际文件
- 压缩示例，消除冗余

### 结构模式
```markdown
---
name: skill-name
description: 当 [具体触发条件] 时使用
---

# Skill 名称

## 0. 快速入门（一条规则 + 决策问题）
## 1. 核心契约（≤7 条必须规则，附验证方式）
## 2. 路由/决策流程
## 3. 失败协议
## 4. 附录（示例、反模式）
```

### CSO（Claude 搜索优化）
- Description = 何时使用，不是做什么
- 关键词：错误、症状、工具
- 禁止在 description 中写工作流摘要（会导致代理走捷径跳过完整 skill）

## 代码风格

**风格定义**：精简、高效、零冗余。

- **注释**：解释「为什么」，而非「是什么」
- **定向变更**：不影响无关章节
- **先读后改**：修改前始终先读取当前内容
- **验证列**：每条必须规则需要可验证标准

## 审查流程


**任何 skill 修改后：**

1. 检查隐藏问题、AI 盲点、过度工程
2. 迭代直到通过

### 审查标准

| 标准 | 目标 |
|------|------|
| 认知负载 | ≤7 条核心契约规则 |
| 可验证性 | 每条规则有验证方法 |
| 决策流程 | 无歧义分支（无「看情况」） |
| 失败协议 | 显式恢复步骤 |
| Token 效率 | 频繁加载的 < 200 词 |

## 常见操作

### 修改 Skill

1. 读取当前 SKILL.md 内容
2. 识别要修改的具体章节
3. 做定向变更（不要重写整个文件）
4. 运行审查验证
5. 应用审计发现的修复

### 添加新示例

- 保持示例最小化（一个优秀的 > 多个平庸的）
- 展示完整可运行的代码
- 解释为什么，而非是什么
- 放在附录章节

### 添加新规则

- 必须在 7 条核心契约限制内
- 必须有可验证标准
- 不得与现有规则重叠
- 考虑：这真的是必须的还是只是建议？

## 反模式

| 反模式 | 为什么不好 |
|--------|-----------|
| Description 中写工作流 | 代理走捷径，跳过完整 skill |
| >7 条核心契约规则 | 认知过载 |
| 「通过 LLM 验证」 | 不可机械验证 |
| ASCII 图表 | 浪费 token，LLM 按序处理文本 |
| 多个类似示例 | 稀释质量，维护负担 |
